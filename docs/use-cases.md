# Workflow Use Case Ideas

A starter list of processes worth encoding as a Progi workflow. Bring your own, or start here.

1. **Writing a blog post** — research, outline, draft, edit, publish checklist
2. **Code review process** — read diff, check conventions, write feedback, request changes or approve
3. **Release management** — bump version, update changelog, tag, draft release notes, notify
4. **Bug triage** — reproduce, identify root cause, write issue, assign priority
5. **Onboarding a new project** — read codebase, document architecture, set up local env, run tests
6. **Weekly planning** — review backlog, set goals, break into tasks, time-block calendar
7. **Content repurposing** — take a blog post, turn it into a Twitter thread, LinkedIn post, and newsletter blurb
8. **Incident response** — detect, assess severity, mitigate, document, post-mortem
9. **Dependency upgrade** — audit outdated packages, check changelogs, upgrade, test, PR
10. **Interview prep** — pick a role, generate questions, mock answers, review gaps
